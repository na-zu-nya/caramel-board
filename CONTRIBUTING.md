# Contributing to Caramel Board

Thanks for your interest in Caramel Board. This is a source-available project
maintained primarily by the project owner. We welcome clear reports and focused
contributions, but review capacity is limited.

## Issues

- Bug reports and feature requests are welcome through GitHub Issues.
- Include reproduction steps, environment details, and expected/actual behavior.
- For security-sensitive reports, do not publish exploit details in a public
  issue. Contact the maintainer privately first.

## Pull Requests

- Please open an Issue or start a discussion before sending a non-trivial PR.
- Unsolicited large PRs may be closed without review.
- Use your own fork. Do not create branches in the upstream repository unless
  you are a maintainer.
- Keep changes focused and consistent with the existing architecture.
- Add tests or update documentation when the change affects behavior.
- UI component changes must include the corresponding Storybook update.

## License of Contributions

By submitting a contribution, you agree that your contribution may be used by
the Caramel Board project under the repository's license, currently the
Caramel Board Source Available License 1.0, or a later license chosen by the
project maintainer.

You also confirm that you have the right to submit the contribution and that it
does not knowingly include code, assets, or other material under incompatible
terms.

## Forks and Builds

Public forks are allowed for development, review, experimentation, and
contribution. A fork must not publish release builds, installers, app bundles,
container images, hosted services, or substantially similar products for
third-party use without written permission from the maintainer.

## Conduct

Be respectful and constructive. The maintainer may limit, close, or moderate
discussions when necessary to keep the project manageable.
